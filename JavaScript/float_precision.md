## 浮点数精度问题

因为浮点数在计算机内是以二进制存储和计算的，所以在浮点数上计算会存在精度问题如：

```javascript
0.1 + 0.2 = 0.30000000000000004;
1.0 - 0.9 = 0.09999999999999998;
```

- 使用 toFixed 进行“四舍五入”
- 将数扩大至整数，再进行计算
- 使用例如 number-precision 等第三库进行计算

**值得注意的是**

toFixed 并不是真正的四舍五入。而是使用更公平的银行家舍入法。具体规则是：

> 四舍六入五考虑，五后非零就进一 \
> 五后为零看奇偶，五前为偶应舍去，五前为奇要进一
